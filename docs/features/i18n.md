# Feature: UI string localization

All static interface text lives in **one dictionary**, keyed, with a block per
locale ‚Äî the same idea as .NET `.resx` files per culture. Components never hold
hardcoded copy; they look strings up by key, and the active locale chooses the
block. To change or translate any UI text, edit `src/i18n/ui.ts` only.

This is **separate from CV-driven content** (About bio, Experience, contact),
which stays dynamic from the PDF (see [cv-content.md](cv-content.md)). Two
sources by design: **i18n files ‚Üí UI chrome**, **CV ‚Üí personal data**.

## Mechanism

- `src/i18n/ui.ts` ‚Äî `ui = { en: {...}, pt: {...} }`, dot-namespaced keys
  (`nav.about`, `hero.tagline`, `activity.stat.current`, ‚Ä¶). `en` is canonical:
  every key a component may request, and `pt` mirrors it. `as const` makes keys
  literal types.
- `src/i18n/utils.ts` ‚Äî `useTranslations(locale)` returns a typed `t(key)`.
  - Key type derives from the `en` block, so a typo or missing key is a
    **compile error** (like resx's generated accessors).
  - Unknown/undefined locale ‚Üí `defaultLang`. A `pt` value missing for a key
    falls back to the `en` string, never blank.

## Usage

`.astro` components read the locale from the router:

```astro
import { useTranslations } from '../i18n/utils';
const t = useTranslations(Astro.currentLocale);
...
<a>{t('nav.about')}</a>
```

**Islands (Preact)** can't read `Astro.currentLocale` at runtime, so the shell
passes `locale` as a prop and the island calls `useTranslations(locale)` itself
(the dictionary is bundled, so it works on both server render and after
hydration). See `GitHub.astro`/`Activity.tsx` and `Projects.astro`/
`ProjectsCarousel.tsx`.

## Conventions

- **Translate** prose, labels, a11y strings (aria-labels, captions).
- **Keep identical across locales** (still keyed, for single-source) the terminal
  idiom: shell commands (`$ git log‚Ä¶`), proper nouns, tech tokens (`zsh`,
  `Arch Linux x86_64`), filenames.
- **Runtime-composed strings** (e.g. "synced 2m ago", the heatmap caption with a
  count) are split into parts (`activity.synced`, `activity.ago.m`) and assembled
  in code, since a number is only known at runtime.
- Page `<title>`/description default from the dictionary in `BaseLayout`
  (`meta.title`, `meta.description`); a page can override via the prop.

## Adding a locale

Add a block to `ui` (e.g. `es: {...}`) mirroring `en`, add it to `languages` and
the `astro.config.mjs` `locales`, add a `src/pages/<loc>/index.astro` mirror, and
provide the matching `public/CV_<LOC>.pdf` for the CV-sourced content.
